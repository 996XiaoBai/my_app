# 测试平台双主题工业 AI 控制台设计

日期：2026-03-19

## 背景

当前测试平台已经具备浅色/深色切换能力，但整体视觉仍停留在“普通后台”的层级：

- 浅色模式专业感不够稳定，存在马卡龙色偏多、信息对比度不够的问题。
- 深色模式虽然更有氛围，但还没有形成真正的“工业 AI 控制台”辨识度。
- 首页工作台、测试用例页、右侧辅助信息区域之间缺少统一的结构语言。

本次设计目标不是简单换色，而是把产品塑造成：

- 默认浅色：专业、稳定、适合长时间办公与评审
- 深色主打：工业控制台、AI 协作中枢、极简高密、强秩序感

## 已确认方向

用户已确认以下方向：

- 默认仍然是浅色专业风
- 深色主题做成更强的主打风格
- 风格参考偏“工业科技控制台”，而不是炫技海报
- 优先保留三栏工作台结构：
  - 左侧导航
  - 中央主任务区
  - 右侧详情/协作辅助栏
- 深色主题强调：
  - 黑灰底
  - 细描边
  - 高对比排版
  - 少彩色、少装饰、更多结构和秩序

## 设计原则

### 1. 双主题不是换色，而是双气质系统

浅色主题承担“效率与可读性”，深色主题承担“品牌识别与 AI 控制台气质”。

- 浅色主题：减少情绪化氛围，提升阅读效率
- 深色主题：加强工业面板感、协作中枢感、智能系统感

### 2. 工业化控制台优先于传统后台

整体应更接近 issue / agent / control center，而不是营销站或低幼 BI 面板：

- 强边界
- 强结构
- 强信息优先级
- 弱装饰

### 3. 高密信息感建立在“秩序”之上

不是堆更多内容，而是通过一致的：

- 栅格
- 字号层级
- 描边规则
- 状态语义

来获得密度感和专业感。

## 视觉系统

## 配色策略

### 浅色主题

关键词：冷静、专业、清晰、可长时间阅读

- 背景主色：冷灰白、蓝灰白
- 卡片底色：纯白或极浅蓝灰
- 主文字：深石墨蓝
- 次级文字：蓝灰中性色
- 主强调色：深空蓝
- 辅助强调色：电光青
- 状态色：
  - 完成：稳重绿
  - 进行中：蓝
  - 未开始：中灰

要求：

- 禁止大面积淡紫、淡绿、淡粉并列使用
- 关键数字和标题必须高对比
- 边框比当前版本更清晰

### 深色主题

关键词：工业、控制台、AI 协作中枢、极简

- 背景主色：黑、深蓝黑、石墨黑
- 面板色：深灰蓝、深炭黑
- 边框色：低亮度冷灰边线
- 主文字：高亮冷白
- 次级文字：冷灰蓝
- 主强调色：深空蓝
- 次强调色：电光青
- 高亮只出现于：
  - 当前任务
  - 进行中状态
  - 关键指标
  - 可执行主按钮

要求：

- 不做大面积霓虹
- 发光只作为“状态控制信号”
- 通过细描边、分区、明暗差建立科技感

## 材质与容器

### 卡片

- 圆角统一缩小到 10px 至 14px 区间
- 取消过于柔软的大圆角后台感
- 使用更轻的软阴影
- 以描边分区为主，阴影为辅

### 容器层级

建议统一三层：

- 页面层：主背景
- 分区层：主卡片 / 工作区
- 控件层：按钮、标签、步骤块、输入框

深色主题里，容器层级通过：

- 深浅不同的深灰
- 细线描边
- 局部高光

而不是彩色底块区分。

## 字体与排版

### 中文

优先现代系统中文栈，保持专业工具气质：

- `PingFang SC`
- `Hiragino Sans GB`
- `Microsoft YaHei`
- 现有系统字体回退

### 数字与英文

核心指标、编号、任务 ID、状态码建议保持更强的工程感：

- 使用 Inter / JetBrains Mono / 现有等宽栈组合

### 层级规范

- 页面主标题：更大、更重
- 模块标题：中大号、明确分区
- 辅助说明：更浅、更短
- 状态与标签：小号、紧凑、语义清晰

目标是让用户第一眼就能看懂：

- 当前任务是什么
- 当前阶段是什么
- 下一步要做什么
- 协作区在右侧哪里

## 结构设计

## 总体骨架

统一采用三栏工作台结构：

### 左栏：导航

职责：

- 一级模块入口
- 模式切换
- 系统设置

风格：

- 浅色：浅灰蓝底
- 深色：深石墨黑底
- 当前项用细高亮边线或高对比底色标记

### 中栏：主任务区

职责：

- 当前任务标题
- 阶段流转
- 主要输入/输出内容
- 核心数据与操作

风格：

- 这是视觉焦点区
- 首页与测试用例页都应共享同一结构语言

### 右栏：详情 / 协作区

职责：

- 任务详情
- 上下文
- 风险摘要
- AI / Copilot / Agent 辅助面板

风格：

- 比中栏更窄
- 更像“辅助控制台”
- 在深色模式下承担最强的工业协作感

## 核心组件方向

## 1. 首页工作台

需要从当前的“卡片堆叠”升级成“控制台”。

改造重点：

- 标题区更像任务总览头部
- 阶段流转区由离散方块改成更线性的任务轨道
- 指标卡降低彩色底块比例，强调结构与数字
- 右侧“专家工具”升级为协作 / 辅助控制台

## 2. 测试用例页

需要从“单页表单 + 结果板”升级成“工业任务工作区”。

改造重点：

- 左侧输入配置区更像参数面板
- 中央结果区保持主输出地位
- 右侧可扩展为详情 / 协作 / agent panel
- 标签、策略按钮、步骤状态统一到新的双主题系统

## 3. 右侧协作面板

新增或强化“issue / agent / copilot”式侧栏语义：

- 当前任务元信息
- 风险状态
- AI 建议
- 关联上下文
- 快速操作

这个区域在深色主题下要最像“工业控制台中的副驾驶席”。

## 4. 流程进度组件

从现在的块状步骤改为更流式、更连续的任务进度表达。

可选实现方向：

- 横向轨道式步骤流
- 圆点 + 连接线
- 带状态信号的线性节点流

要求：

- 完成、进行中、未开始三态区分必须非常清楚
- 深色模式下允许弱发光
- 浅色模式下必须克制且高可读

## 5. 指标卡

从“彩色信息块”升级为“控制台指标模块”。

要求：

- 数字更大更醒目
- 标签更轻
- 趋势线作为辅助，不抢主体
- 色彩不靠大面积底色，而靠局部强调

## 实施范围

第一阶段建议优先覆盖：

- 顶部导航
- 左侧导航
- 首页工作台
- 测试用例页
- 通用卡片、按钮、标签、输入框
- 阶段流转组件
- 右侧详情 / 协作侧栏基础骨架

第二阶段再扩展到：

- 周报页
- 需求评审页
- 流程图页
- 通用模块页

## 实施策略

建议按以下顺序实施：

1. 先重构主题 token
2. 再统一基础容器与排版
3. 再改首页工作台骨架
4. 再改测试用例页骨架
5. 最后补右侧协作侧栏与其他模块

这样可以避免每个页面单独修色，造成系统不一致。

## 风险与注意点

- 深色主题需要强气质，但不能影响表格、输入框、长文本可读性
- 工业科技感必须建立在“信息清晰”之上，而不是背景特效
- 右侧协作栏不能只是装饰，需要预留真实功能承载空间
- 双主题应共享同一组件语义，不应发展为两套完全不同的系统

## 验收标准

设计完成后，至少满足以下结果：

- 浅色主题看起来像专业工程平台，而不是彩色后台
- 深色主题看起来像 AI 工程控制台，而不是普通 dark mode
- 首页能体现“任务中枢”而不是静态仪表盘
- 测试用例页能体现“工作区”而不是普通表单页
- 右侧协作面板具备明显的 issue / copilot / agent 辅助感

## 当前结论

本次设计已确认：

- 采用双主题系统
- 默认浅色专业风
- 深色主题作为更强的主打风格
- 视觉方向采用工业科技控制台路线
- 结构采用三栏工作台
- 右侧保留协作 / agent panel 能力

下一步在用户确认本设计文档后，再进入真实实现。
