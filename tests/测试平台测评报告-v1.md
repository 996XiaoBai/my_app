- `周报生成`：功能设计有价值，但目前受外部依赖稳定性影响，尚不适合作为稳定生产能力。

### 当前平台真实状态
- 不是“所有功能都不可用”
- 也不是“所有功能都已经完成验收”
- 更准确的定位是：后端能力已初步成型，平台壳和外部依赖韧性仍需补强

## 衡量方法建议
后续每个版本建议固定输出以下指标：

1. 功能通过率  
   `通过功能点 / 总功能点`

2. 核心链路成功率  
   `成功调用次数 / 总调用次数`

3. 页面冒烟通过率  
   `可正常打开并完成首个动作的关键页面数 / 关键页面总数`

4. 外部依赖韧性  
   `成功返回 + 优雅降级返回 / 总调用次数`

5. AI 输出质量均分  
   从完整性、准确性、可执行性、一致性四项评分

6. 回归覆盖度  
   后端单元 + 接口 + 前端组件 + 页面 E2E 的覆盖情况
