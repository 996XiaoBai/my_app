## 已真实实测功能
### 1. 需求评审
- 测试方式：真实启动 FastAPI 后端后调用 `/run`
- 输入：

```text
mode=review
requirement=用户可以使用手机号和验证码登录系统
```

- 预期：
  - 返回 `success=true`
  - 返回评审报告正文
  - 返回结构化风险和动态洞察
  - 不出现服务端异常
- 实际：
  - 返回 `success=true`
  - 返回完整 Markdown 评审报告
  - 返回结构化 `findings_json`
  - 返回动态洞察：聚焦验证码链路、频控、幂等与资损风险
- 结果判定：通过
- 风险判断：
  - 功能可用性较好
  - 仍受外部大模型可用性影响
- 结论：
  - 当前属于平台中最接近可直接落地使用的核心能力之一

### 2. 测试数据准备
- 测试方式：真实启动 FastAPI 后端后调用 `/run`
- 输入：

```text
mode=test-data
requirement=CREATE TABLE user_account (id BIGINT PRIMARY KEY, user_name VARCHAR(64), mobile VARCHAR(20), balance DECIMAL(10,2), created_at DATETIME);
```

- 预期：
  - 返回 `success=true`
  - 识别表结构
  - 生成 `SELECT/INSERT/UPDATE/DELETE`
  - 生成 Markdown 与 SQL 文件内容
- 实际：
  - 返回 `success=true`
  - 成功识别 `user_account`
  - 生成完整 SQL 模板
  - 返回 Markdown 和 `sql_file_content`
  - 返回动态洞察，指出 `balance` 字段的越权修改和资损测试风险
- 结果判定：通过
- 风险判断：
  - 规则型能力较稳定
  - 动态洞察依赖大模型，但主结果生成不完全依赖大模型
- 结论：
  - 当前是平台中稳定性和实用性都较高的能力

- 测试方式：

- 风险判断：

### 4. 周报生成
- 测试方式：真实启动 FastAPI 后端后调用 `/run`
- 输入：

```text

```

- 预期：
  - 返回 `success=true`
  - 生成 Markdown 周报
  - 在关闭飞书发布时仍可返回本地结果
- 实际：

- 风险判断：
  - 高风险
  - 当前对上游依赖限流不够稳健
- 结论：
  - 功能设计有价值，但在真实环境中不能视为稳定能力
