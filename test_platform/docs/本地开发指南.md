# 测试平台本地开发指南

## 1. 推荐环境

- 操作系统：macOS
- Python：`3.11`
- Node.js：使用当前仓库已安装版本即可
- 推荐虚拟环境：`test_platform/.venv`

仓库已经通过以下方式固定了 Python 基线：

- 根目录 `.python-version` 声明为 `3.11`
- `test_platform/scripts/resolve_python.sh` 会优先选择满足 `3.11+` 的解释器
- `make ui`、`start.sh`、`start_web_ui.sh` 都会优先走测试平台的 `3.11` 环境

## 2. 首次进入项目

推荐先激活测试平台专用虚拟环境：

```bash
source ~/.zprofile
source test_platform/.venv/bin/activate
python3 -V
```

预期输出应为 `Python 3.11.x`。

如果只想直接启动项目，也可以不手动激活虚拟环境，直接使用仓库脚本。它们会自动解析可用解释器。

## 3. 常用启动方式

### 3.1 Streamlit 单体模式

适合体验功能最完整的测试平台 UI。

```bash
make ui
```

或者：

```bash
bash start_web_ui.sh
```

默认端口由环境变量 `TEST_PLATFORM_STREAMLIT_PORT` 控制，未配置时使用 `8501`。

### 3.2 Web 工作台模式

适合体验 FastAPI + Next.js 的新版工作台。

```bash
./start.sh
```

该脚本会：

1. 启动 FastAPI，端口 `8000`
2. 启动 Next.js，端口 `3000`
3. 退出前端进程时自动回收后端进程

## 4. 常用验证命令

### 4.1 Python 平台测试

```bash
source test_platform/.venv/bin/activate
python3 -m pytest tests/platform_tests -q
```

### 4.2 前端静态检查

```bash
cd test_platform/web
npm run lint
```

### 4.3 前端原生测试

```bash
cd test_platform/web
npm test
```

### 4.4 前端构建验证

```bash
cd test_platform/web
npm run build
```

## 5. 当前入口关系

当前测试平台已经做过一轮入口收口，建议按下面的理解使用：

- `test_platform/app.py`：Streamlit 主入口兼容壳
- `test_platform/ui/main_streamlit_app.py`：真实的 Streamlit UI 实现
- `test_platform/ui/streamlit_app.py`：旧入口兼容壳
- `test_platform/api_server.py`：FastAPI 接口服务
- `test_platform/web`：Next.js 前端工作台

## 6. 常见问题

### 6.1 为什么我看到的 `python3` 还是旧版本？

先执行：

```bash
source ~/.zprofile
python3 -V
```

如果你是旧开的终端窗口，重新打开一个新的终端标签页也可以。

### 6.2 为什么 `python3 -m pytest` 提示没有安装 `pytest`？

这是因为项目依赖安装在虚拟环境里，而不是系统 Python 里。先激活：

```bash
source test_platform/.venv/bin/activate
```

然后再执行测试命令。

### 6.3 当前最推荐的工作流是什么？

```bash
source ~/.zprofile
source test_platform/.venv/bin/activate
python3 -V
python3 -m pytest tests/platform_tests -q
make ui
```
